1. Sofia's Comment Replies

This is the single largest automation opportunity on the team. Sofia spends 3-5 hours every day manually processing 70-240 social media comments through Hootsuite, one at a time, using Google Gemini as an AI assistant via copy-paste.

What We Found

HIGH CONFIDENCE▶

Sofia's daily workflow, captured directly from keystroke data:

  1. Opens Hootsuite. Sees 70-240 items in the inbox (rising trend — Mar 4 hit 241, the highest observed, coinciding with spring campaign launch).

The Case That Automation Would Be Better

KEY ARGUMENT▶

This isn't just about saving time. A properly configured pipeline would produce better replies than the current process. Here's why:

Sofia currently has no system prompt. Every Gemini conversation starts fresh. No brand voice instructions. No return policy details. No CS routing rules. No platform context. She's getting generic AI output and then manually fixing it every single time.

43% of her edits are adding information a system prompt would include automatically. Things like:

  • The support email: hello@meridianhomeco.com
  • Return window: "30-day hassle-free returns"
  • Free shipping threshold: "Free shipping on orders over $75"
  • Product care links and collection page URLs

A system prompt with brand voice, return policy, free shipping threshold, CS routing rules, and platform metadata (which Hootsuite provides via API) would include all of this automatically. The first drafts would be better than what Gemini gives Sofia cold.

Consistency improves. A pipeline doesn't get tired at comment #118. It doesn't forget to include the return window on the 82nd reply. It doesn't gradually shorten replies as the day wears on.

Here's how fast this gets good: We recommend starting with a proof run: 20 recent comments, brand-voice replies generated, you review. 5 minutes to validate. We adjust and send another 20. Three rounds of this — 15 minutes total — and the system is calibrated to your brand. After that, the pipeline runs. Sofia reviews exceptions, not the whole inbox.

Sofia grew your social engagement with a seasonal content strategy she built herself. That's the kind of work Sofia should be doing — content planning, influencer relationships, platform optimization. Not copy-pasting the return policy into Gemini 118 times a day.

3. Maya's Sold-Out Products

What We Found

99% CONFIDENCE▶

Maya searches Meridian's Shopify catalog by typing individual product names — one at a time — into the search bar. 998 product name searches in 30 days. 622 unique product names. She opens each product page, reads the inventory status, and records it in a spreadsheet for the merchandising team.

The problem: Shopify already has a filter for this. You can filter products by inventory_quantity = 0 and get the complete list in seconds. She's searching 620+ products by name because she either doesn't know the filter exists, or the spreadsheet list she's working from drives her to look them up individually.

Implementation — 20 minutes, zero code

Go to Shopify Admin → Products
Click "More filters" → select "Inventory" → set to "is equal to 0"
Click "Export" to download the complete list as CSV
Open in Google Sheets. Done.

For an automated version:

GET /admin/api/2024-10/products.json?status=active&fields=id,title,variants
# Then filter: variant.inventory_quantity == 0

# Or use GraphQL for a single call:
{
  products(first: 250, query: "inventory_total:0") {
    edges { node { title status totalInventory variants(first:5) {
      edges { node { title inventoryQuantity } }
    }}}
  }
}

Scheduled weekly (or triggered after each restocking run), this eliminates the entire 11 hours/month of manual product lookups.

How It Was Analyzed

Analysis was performed locally via a secure Postgres pipeline on Go2's infrastructure. Raw data never left the local environment. Only derived insights and findings were compiled into this report. Every data file was loaded and cross-referenced. Keystroke sequences were decoded to reconstruct actual workflows — what keys were pressed, what windows were active, what text was typed. App window titles were parsed to identify specific tools, pages, and documents. ShopifyQL queries were extracted from browser URL parameters. AI prompt patterns were reconstructed from typing sequences. All numbers in this report trace back to counted events in the source data.

Data Collection

All behavioral data was captured by the Cowork.ai telemetry daemon running within each team member's Go2 desktop application. The daemon captures the full behavioral data spectrum — application usage, window focus patterns, keystroke cadence analysis, URL navigation sequences, idle detection thresholds, and session boundary events — all stored locally on each team member's machine before secure insight extraction.

Known Data Limitations

  • macOS telemetry coverage: The Cowork.ai daemon on macOS captures app-level and window-level events but has reduced keystroke cadence telemetry compared to the Windows agent. Maya and David (both macOS) were analyzed primarily through application focus patterns and URL navigation data.
  • Cadence binning: Keystroke telemetry is captured in 1-minute cadence bins, not individual keypress events. Rapid sequences within a single bin are collapsed. All keystroke-derived counts in this report are floor estimates.
  • Browser opacity: When a team member works entirely within Chrome, the telemetry daemon captures the active window title but not always the full URL path. Site-level behavioral data is incomplete for some team members on certain days.
  • Self-reported metrics: Ticket counts, satisfaction scores, and call volumes in team activity logs are self-reported and were not cross-validated against source system APIs for this audit.
  • David: partial daemon deployment. The Cowork.ai agent appears to have been partially configured on David's machine for most of the audit period. Behavioral data coverage is insufficient to assess his workflows or throughput with confidence.
