Deployment
5-Step Deployment Process
Total time: under 15 minutes. No dedicated infrastructure. No firewall changes. No VPN required.
1
Configure SSO
3 minutes
SAML 2.0 and OIDC supported. Works with Okta, Azure AD, Google Workspace, OneLogin, and any SAML-compliant IdP.
# Okta SAML Configuration SSO URL: https://auth.go2.io/saml/acs Entity ID: https://go2.io/sp NameID: email Attributes: email, firstName, lastName, department # Or use OIDC Redirect URI: https://auth.go2.io/oidc/callback Scopes: openid email profile
2
Enable SCIM Provisioning
2 minutes
SCIM 2.0 for automated user provisioning and deprovisioning. Assign Go2 to groups in your IdP and users are automatically created.
# SCIM Endpoint Base URL: https://api.go2.io/scim/v2 Token: [generated in Go2 admin panel] # Supported operations POST /Users # Create user PATCH /Users/{id} # Update user DELETE /Users/{id} # Deprovision user GET /Groups # List groups
3
Deploy Desktop Agent
5 minutes
Lightweight agent (12MB). Available via MSI, PKG, or MDM-compatible formats. Silent install supported. Auto-updates.
# MDM deployment (Jamf, Intune, Mosyle) curl -sL https://dl.go2.io/agent/latest/go2-agent.pkg # Windows (GPO/SCCM) msiexec /i go2-agent.msi /quiet ORG_KEY=your-key # macOS (Homebrew) brew install --cask go2 # Linux (apt) sudo apt install go2-agent

Agent specs: <1% CPU, ~48MB RAM, auto-updates, user-pausable.

4
Configure Integrations
3 minutes
OAuth-based integrations with Slack, Jira, GitHub, Salesforce, and 40+ tools. No API keys to manage.
# Webhook for SIEM integration POST https://api.go2.io/v1/webhooks { "url": "https://your-siem.com/ingest", "events": ["audit.login", "audit.data_export", "audit.config_change"], "format": "syslog" }
5
Set Policies & Go Live
2 minutes
Configure data retention, privacy settings, and role-based access from the admin panel. Default policies are privacy-first.
# Default privacy policy (can be customized) { "keystroke_logging": false, "screen_recording": false, "url_tracking": "domain_only", "personal_apps": "excluded", "data_retention": "90_days", "employee_dashboard": true, "employee_pause": true }

FAQ
Common IT Questions
What data does the agent collect? +
The agent captures application usage patterns, active/idle time, and workflow sequences. It does NOT log keystrokes, record screens, capture clipboard content, or monitor communication content. URL tracking is domain-level only (not full URLs). Personal apps can be excluded entirely. Employees can view all their collected data and pause the agent at any time.
Where is data stored? +
Data is stored in AWS with your choice of region: US-East, US-West, EU-Frankfurt, or APAC-Sydney. Data never leaves your selected region. AES-256-GCM encryption at rest with customer-managed keys available. 90-day default retention (configurable). Full GDPR Article 17 right-to-deletion support.
What are the system requirements? +
Windows 10+ (x64), macOS 12+ (Intel/Apple Silicon), Ubuntu 20.04+ / RHEL 8+. Agent size: 12MB. Runtime: <1% CPU, ~48MB RAM. No kernel extensions, no root access required. Auto-updates with rollback. Compatible with all major MDM solutions (Jamf, Intune, Mosyle, Workspace ONE).
Does it work with VDI environments? +
Yes. Tested with Citrix, VMware Horizon, Amazon WorkSpaces, and Windows 365. Non-persistent VDI supported with machine-level configuration. Agent can be included in golden images.
What if an employee leaves? +
SCIM deprovisioning automatically disables the user account and stops data collection. Data is retained per your policy (default 90 days), then permanently deleted. The agent can be remotely wiped via MDM or auto-uninstalls when the user is deprovisioned.
What about network impact? +
Minimal. The agent batches data locally and syncs every 5 minutes over TLS. Typical bandwidth: <50KB/minute per user. Edge processing reduces data volume by 90% before transmission. No impact on application performance.
How do we handle the security questionnaire? +
We pre-fill SIG Lite and CAIQ questionnaires. Most IT security reviews complete in 1-2 days. We provide: SOC 2 Type II report, penetration test summary, architecture diagram, data flow documentation, and a dedicated security engineer for any questions.
