Go2 REST API v3

Integrate Go2 productivity data into your workflows. RESTful, JSON-based, and built for developers.

Base URL: https://api.go2.io/v3

Authentication

All API requests require authentication via Bearer token. Generate API keys from your Go2 admin dashboard.

Header Format

Authorization: Bearer go2_sk_live_a1b2c3d4e5f6...

API keys start with go2_sk_live_ (production) or go2_sk_test_ (sandbox).

Example RequestcURL
curl -X GET https://api.go2.io/v3/analytics \
  -H "Authorization: Bearer go2_sk_live_a1b2c3..." \
  -H "Content-Type: application/json"

Rate Limits

Default Limits

Free: 100 requests/min • Business: 1,000 requests/min • Enterprise: 10,000 requests/min

Rate limit headers are included in every response:

X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 847
X-RateLimit-Reset: 1710684000

Analytics

Retrieve aggregated productivity analytics for your organization, teams, or individual users.

GET /analytics/summary

Returns aggregated analytics for the specified time period.

ParameterTypeRequiredDescription
start_datestringRequiredStart date (ISO 8601, e.g., 2026-03-01)
end_datestringRequiredEnd date (ISO 8601)
team_idstringOptionalFilter by team
granularitystringOptionaldaily, weekly, monthly (default: daily)
Response200 OK

  "data": {
    "period": { "start": "2026-03-01", "end": "2026-03-17" },
    "summary": {
      "total_active_hours": 12840.5,

      "total_users": 142,
      "top_categories": [
        { "name": "Development", "hours": 4280.2, "pct": 33.3 },
        { "name": "Communication", "hours": 3210.1, "pct": 25.0 }
      ]
    }
  },
  "meta": { "request_id": "req_7f3k..." }
}

Webhooks

POST/webhooks

Register a webhook endpoint to receive real-time events.

RequestJSON
{
  "url": "https://your-app.com/go2-webhook",
  "events": ["insight.created", "alert.triggered", "user.joined"],
  "secret": "whsec_your_signing_secret"
}

Webhook payloads are signed with HMAC-SHA256 using your webhook secret. Verify the X-Go2-Signature header on every request.

Data Exports

POST/exports

Request a data export. Returns a job ID — poll the status endpoint until complete.

RequestJSON
{
  "format": "csv",
  "start_date": "2026-01-01",
  "end_date": "2026-03-17",
  "include": ["analytics", "insights"]
}
GET/exports/{export_id}

Check export status and download URL.
