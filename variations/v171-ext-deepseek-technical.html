This document outlines the architecture, data pipeline, AI models, and privacy design of the Go2 system.

1. Architecture Overview

Go2 installs a lightweight desktop agent (~15MB RAM) on employee machines. The agent captures keystrokes, audio (via ambient sampling), screenshots (periodic), application usage, and workflow patterns.

System Architecture

Local Agent

Electron-based, encrypted local storage, offline queue.

→

Edge Processing

Data anonymization, feature extraction, compression.

→

AI Pipeline

Claude 3, GPT-4, Gemini Pro — multi-model routing.

→

Insights Engine

Coaching tips, aggregated metrics, workflow maps.

Data flow: Raw capture → Local encryption → Edge processing → AI processing → Insights generation → User-facing outputs.

2. Data Pipeline

Every captured event undergoes a 5-step pipeline before any AI processing.

1

Local Capture

Event-driven hooks capture keystrokes (as semantic sequences), audio snippets (5-sec ambient samples), screen frames (2/min), app switches.

2

Encryption & Queue

Data encrypted with AES-256-GCM, stored locally in a FIFO queue. Only metadata (timestamps, event types) sent for routing.

3

Edge Processing

On our edge servers: anonymization (ID stripping), feature extraction (e.g., "coding session", "meeting"), compression.

4

AI Model Routing

Based on event type: Claude for coaching, GPT for workflow mapping, Gemini for cross-validation. Models never receive raw PII.

5

Output Generation

AI outputs stored in user-specific buckets. Workers see full outputs; managers see aggregates (e.g., "Team spends 23% time in meetings").

3. Privacy Architecture

Privacy is built into the pipeline, not added as a filter.

Worker Visibility

Every worker accesses a personal dashboard showing all their own raw data, AI coaching tips, and workflow maps. No data hidden.

GET /api/v1/user/raw-events

Manager Visibility

Managers see only aggregated metrics: percentages, trends, anonymized patterns. Never individual keystrokes, audio, or screens.

GET /api/v1/team/aggregates

AI Processing

AI models receive anonymized, feature-extracted data. They output coaching and maps, but cannot reconstruct original PII.

POST /api/v1/ai/coaching

6. Deployment & Scale

Built on 7+ years of staffing data, ~1M workers measured.

  • Agent deployment: Silent install via Docker, MSI, or pkg.
  • AI model fallback: If Claude fails, GPT-4 backs up.
  • Data retention: Raw data stored locally for 30 days, anonymized aggregates kept indefinitely.
  • Scale tested: Up to 10,000 concurrent agents.

Target buyer: SMB owner with 15–50 remote employees.
