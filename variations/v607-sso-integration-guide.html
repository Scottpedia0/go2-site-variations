SAML 2.0 SP-initiated flow with signed assertions

1 Create Go2 App in Okta

Log into your Okta Admin Console and create a new SAML 2.0 application.

  • ☐Go to Applications → Create App Integration
  • ☐Select SAML 2.0 as the sign-on method
  • ☐Name: "Go2 Productivity"
  • ☐Logo: Download from Go2 admin → SSO → Assets

2 Configure SAML Settings

Enter the following SAML configuration values in the Okta app setup wizard:

FieldValue
Single Sign-On URLhttps://auth.go2.io/saml/acs
Audience URI (SP Entity ID)https://auth.go2.io/saml/metadata
Name ID FormatEmailAddress
Application UsernameEmail
ResponseSigned
Assertion SignatureSigned
Signature AlgorithmRSA-SHA256
ⓘ

Your unique tenant ID will be provided in the Go2 admin console under Settings → SSO. Replace the base URL if you're on a custom domain.

4 Complete Setup in Go2

Copy the Identity Provider metadata from Okta and paste it into Go2:

  • ☐In Okta: Go to Sign On tab → Copy Identity Provider metadata URL
  • ☐In Go2: Go to Admin → Settings → SSO → Paste metadata URL
  • ☐Click "Test Connection" to verify
  • ☐Enable SSO for your organization
✓

Once enabled, users will be redirected to Okta for authentication. Existing sessions will continue until expiry.

1 Register Go2 in Azure AD

Navigate to Azure Active Directory → Enterprise Applications → New Application → Create your own application.

SettingValue
Identifier (Entity ID)https://auth.go2.io/saml/metadata
Reply URL (ACS)https://auth.go2.io/saml/acs
Sign on URLhttps://app.go2.io/login
Relay State(leave blank)
Logout URLhttps://auth.go2.io/saml/slo

2 Download Federation Metadata

Download the Federation Metadata XML from Azure AD and upload it to Go2 Admin → Settings → SSO.

ⓘ

Azure AD supports both SAML 2.0 and OIDC. We recommend SAML for enterprise SSO and OIDC for developer integrations.

SCIM 2.0 Provisioning

Automate user provisioning and deprovisioning with SCIM. When you add or remove users in your IdP, Go2 updates automatically.

SCIM ConfigurationAll Providers
// SCIM Endpoint
Base URL: https://api.go2.io/scim/v2

// Authentication
Method:   Bearer Token
Token:    (Generated in Go2 Admin → SSO → SCIM)

// Supported Resources
Users:    /scim/v2/Users
Groups:   /scim/v2/Groups

// Supported Operations
CREATE, READ, UPDATE, DELETE, SEARCH
⚠

Important: SCIM provisioning is available on Business and Enterprise plans. The SCIM bearer token should be stored securely and rotated every 90 days.
