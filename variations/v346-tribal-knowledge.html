Production Deployment Process

Auto-generated by Go2
This document was auto-generated by observing 47 deployments by 6 engineers over the past 30 days. Last updated: 2 hours ago.

Overview

The production deployment process typically takes 25-40 minutes and involves the staging environment, automated tests, manual smoke tests, and the production push.

Steps (observed from top performers)

  1. Create PR from feature branch to main. Ensure CI passes (avg: 8 min).
  2. Get at least 2 approvals. Sarah typically reviews architecture, Mike reviews tests.
  3. Merge to main. Wait for staging deploy (automated, ~3 min).
  4. Critical: Run the billing reconciliation check. This step is skipped 30% of the time but causes issues in 80% of skip cases. Sarah always runs it.
  5. Run smoke tests on staging. Check: /health, /api/v2/status, and one manual order flow.
  6. Trigger production deploy via #deploy Slack channel with /deploy prod.
  7. Step 7 (the one everyone asks about): Wait 60 seconds after deploy, then hit /api/cache/warm. This preloads the CDN cache. Without it, first 100 users get 3s load times.
  8. Monitor Datadog dashboard "Prod Health" for 15 minutes. Watch for error rate > 0.5%.

Common Gotchas (learned from incidents)

If deploy fails mid-way, do NOT retry immediately. Check the DB migration status first — a partial migration + retry = data corruption. This happened on March 3rd.
